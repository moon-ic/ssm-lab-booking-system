# 设备借用后台管理系统 PRD

## 1. 背景与目标

本项目用于建设一套面向学校、实验室和教学场景的设备借用后台管理系统，统一管理设备信息、预约审核、借用归还、维修处理、账号权限、消息提醒和统计分析。

产品目标：

- 建立覆盖超级管理员、管理员、教师、学生四类角色的统一后台
- 打通账号创建、预约、审核、借用、归还、维修、消息提醒、统计分析全流程
- 明确角色权限边界，支持后续前后端联调与实施
- 输出稳定的 API 输入，支撑后续接口实现

## 2. 角色定义

### 2.1 超级管理员

- 创建和管理管理员账号
- 管理角色与权限
- 管理菜单与图标配置
- 查看全局统计分析
- 管理设备导入配置

### 2.2 管理员

- 创建教师账号
- 管理设备信息
- 审核预约申请
- 处理维修流程
- 查看统计分析
- 执行设备导入

### 2.3 教师

- 创建与删除学生账号
- 审核学生预约
- 查看学生借用、归还、维修与消息情况

### 2.4 学生

- 查看设备并发起预约
- 确认取用与归还
- 提交维修申请
- 查看个人借阅记录
- 接收并确认消息提醒
- 首次登录后修改密码

- 个人中心

- 预约与审核

## 4. 核心业务规则

### 4.1 账号与密码

- 学生使用学号登录
- 教师使用工号登录
- 管理员与超级管理员使用系统账号登录
- 新建账号默认密码统一为 `000000`
- 所有新建账号首次登录时必须修改密码
- 管理员和超级管理员支持为下级账号重置密码，重置后密码恢复为 `000000`
- 账号支持启用与禁用

### 4.2 预约审核

- 学生提交预约时必须填写设备、预约时间、用途
- 同一设备在时间段冲突时不可重复预约
- 教师和管理员均可完成单级审核
- 审核结果仅包含“通过”与“驳回”
- 驳回必须填写原因
- 审核通过后生成待取用记录并锁定对应设备时间段
- 审核通过后的预约不允许学生主动取消
- 审核通过但在预约开始后 24 小时内仍未取用的预约自动失效

### 4.3 借用归还

- 仅待取用状态可确认取用
- 取用成功后借用记录变为“借用中”，设备状态变为“BORROWED”
- 仅借用人本人可确认归还
- 归还时必须填写归还时间和设备状态
- 设备正常归还后恢复为可用
- 设备异常归还后进入维修流程
- 管理员不代替用户确认取用与归还

### 4.4 到期与过期提醒

- 系统默认在到期前 3 天、到期前 1 天发送提醒
- 超过应归还时间后立即生成过期提醒
- 到期与过期提醒都写入消息中心
- 提醒消息分为“未确认”和“已确认”两种状态
- 用户进入消息中心后可逐条确认消息

### 4.5 待确认消息

- 消息中心统一承载到期提醒、过期提醒、预约失效提醒、首次登录改密提醒、密码重置提醒、维修处理提醒
- 消息分为 `UNCONFIRMED` 和 `CONFIRMED`
- 未确认消息默认按时间倒序展示
- 用户确认后保留消息历史，不物理删除

### 4.6 设备管理与导入

- 每台设备拥有唯一设备 ID 和唯一设备编码
- 管理员与超级管理员均可新增、编辑、停用设备
- 设备导入采用“表单录入 + 图片上传”的方式，不做 Excel 批量导入
- 导入时必须填写设备名称并上传设备图片
- 上传成功后系统自动注册唯一设备 ID 与设备编码
- 导入后的设备默认状态为 `AVAILABLE`

### 4.7 图标库

- 图标库采用系统内置维护方式
- 超级管理员可查看内置图标列表并为菜单配置图标
- 图标库不接入第三方远程资源

### 4.8 统计分析

- 热门设备榜、设备损坏榜、用户违规榜均提供三个维度：
  - 总榜
  - 半年榜
  - 月榜
- 热门设备榜按借用次数排序
- 设备损坏榜按维修申请次数和损坏次数综合排序
- 用户违规榜按逾期次数、损坏归还次数排序
- 统计概览展示当前累计总量，不区分榜单维度

## 7. 验收标准

- 四类角色均可完成各自权限范围内操作
- 新账号首次登录必须修改密码
- 管理员和超级管理员可执行重置密码
- 用户可接收到到期与过期提醒，并能在消息中心确认
- 设备导入支持表单录入名称、上传图片、自动生成唯一标识
- 图标库为内置维护
- 统计榜单支持总榜、半年榜、月榜三种维度
- PRD 不保留待确认项，足以支撑后续接口设计
