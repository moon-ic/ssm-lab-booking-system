# 原则

- 一次只做一个模块
- 先设计，再实现
- 先 mock，再接真实接口
- 严格对齐 API（字段 / 状态 / 权限不改）
- 结构清晰：pages / components / store / api / types / mock
- 每步必须包含自测 + 回归

# 输入

当前模块 + PRD + API

# 输出

## 1. 设计

- 核心业务
- 页面与交互
- 数据模型（type / enum）
- 接口映射（入参 / 出参）
- 状态设计（业务状态 + UI 状态）
- 权限控制（页面 / 按钮）
- 边界与异常
- mock 方案

## 2. 代码

按顺序输出（仅当前模块）：

1. types
2. mock
3. api
4. store（如需要）
5. page / components
6. route（如需要）

要求：

- 可运行最小闭环
- 字段与 API 完全一致
- mock 覆盖：成功 / 失败 / 空 / 权限 / 状态冲突

## 3. 测试

### 自测

- 页面可进入
- 列表 / 表单 / 操作正常
- 状态流转正确
- 错误提示明确
- 权限生效
