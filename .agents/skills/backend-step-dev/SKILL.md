---

# 原则

- 一次只做一个模块
- 先设计，再代码
- 先内存实现，再接 MySQL / Redis
- 严格遵循 API，不修改
- 分层清晰：controller / service / repository
- TODO 单独列出

# 输入

当前模块 + API 文档

# 输出

## 1. 设计

1. 核心业务
2. 数据模型（Entity / DTO / VO / Enum）
3. 接口设计（入参 / 出参）
4. Service 设计（方法 + 流程）
5. 权限设计
6. 边界条件与异常
7. 实现方式（内存）

## 2. 代码

- 文件清单
- 按依赖顺序逐个输出
- 仅当前模块
- 可直接运行
- 字段与 API 一致

## 3. 过程补充

- 已完成内容
- 下一步建议
- 风险 / 待确认项

# 模块示例

认证与登录 → 用户管理 → 角色与权限 → 设备管理 → 预约与审核 → 借用与归还 → 维修管理 → 个人中心 → 统计分析 → 菜单/图标配置 → 定时任务 → Redis → MySQL
